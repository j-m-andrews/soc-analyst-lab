INCIDENT REPORT — PERSISTENCE VIA SCHEDULED TASK & BRUTE FORCE ATTEMPT
Report ID: IR-2026-002 Date: February 25, 2026 Analyst: Jordan Andrews Severity: High Status: Closed

1. EXECUTIVE SUMMARY
On February 25, 2026, suspicious activity was detected on host Windows_10 . A scheduled task named \WindowsUpdateHelper was created using PowerShell with hidden execution flags, indicating an attempt to establish persistent access. Additionally, multiple failed logon attempts were observed against the same host suggesting a brute force attempt. Both activities were detected via Splunk alerting and endpoint telemetry.

3. ATTACK CHAIN
Persistence — T1053.005 (Scheduled Task) A scheduled task named \WindowsUpdateHelper was created on host Windows_10 by account WINDOWS\vboxuser. The task was configured to execute powershell.exe -WindowStyle Hidden on every user logon. The hidden execution flag indicates deliberate attempt to avoid user detection. The task name was chosen to blend in with legitimate Windows Update processes.
Credential Access — T1110 (Brute Force) Multiple failed logon attempts (EventCode 4625) were observed against the host followed by a successful logon (EventCode 4624, Logon Type 2). This pattern is consistent with a brute force or password guessing attack.

7. CONTAINMENT & REMEDIATION
In a real incident the following steps would be taken:
·	Immediately disable and delete the \WindowsUpdateHelper scheduled task
·	Reset credentials for vboxuser and any other potentially compromised accounts
·	Review all scheduled tasks on the host for additional malicious entries
·	Block the source IP of brute force attempts at the firewall
·	Scan endpoint for additional persistence mechanisms (registry run keys, services)
·	Review lateral movement indicators across other hosts

8. ROOT CAUSE
An attacker gained or attempted to gain access to the host via brute force and established persistence through a scheduled task designed to survive reboots and blend in with legitimate Windows processes. The hidden PowerShell execution flag indicates awareness of defensive monitoring.

9. RECOMMENDATIONS
·	Implement account lockout policy after 3-5 failed logon attempts
·	Alert on any scheduled task creation outside of approved change windows
·	Audit all existing scheduled tasks against an approved baseline
·	Deploy MFA to prevent credential-based attacks
·	Consider application whitelisting to prevent unauthorized PowerShell execution
·	Enable PowerShell Script Block Logging for deeper visibility
