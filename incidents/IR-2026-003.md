INCIDENT REPORT — REGISTRY PERSISTENCE & LATERAL MOVEMENT
Report ID: IR-2026-003 Date: February 26, 2026 Analyst: Jordan Andrews Severity: High Status: Closed

1. EXECUTIVE SUMMARY
On February 26, 2026, two suspicious activities were detected on host Windows_10. A registry run key named WindowsSecurityHelper was created to execute hidden PowerShell on every system startup, indicating an attacker establishing persistent access. Shortly after, explicit credential authentication over the network was observed from account vboxuser, consistent with lateral movement activity. Both techniques were detected via Splunk alerting and Sysmon endpoint telemetry.

3. ATTACK CHAIN
Persistence — T1547.001 (Registry Run Keys) A registry run key named WindowsSecurityHelper was added to HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Run via reg.exe. The key was configured to execute powershell.exe -WindowStyle Hidden on every system startup. The key name was chosen to masquerade as a legitimate Windows security component (T1036 — Masquerading).
Sysmon EventCode 13 captured the registry modification:
·	Target Object: HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Run\WindowsSecurityHelper
·	Details: powershell.exe -WindowStyle Hidden -Command 'Write-Host Running'
·	Image: C:\Windows\system32\reg.exe
Lateral Movement — T1021.006 (Remote Services: Windows Remote Management) At 17:56:42, account vboxuser was observed authenticating to the host over the network using explicit credentials (EventCode 4648). This activity was generated via PowerShell remoting (WinRM) and is consistent with an attacker using stolen credentials to move laterally between systems.

7. CONTAINMENT & REMEDIATION
In a real incident the following steps would be taken:
·	Immediately delete the malicious registry run key
·	Reset credentials for vboxuser and audit all accounts for compromise
·	Disable WinRM on endpoints where remote management is not required
·	Review all registry run keys across affected hosts against approved baseline
·	Hunt for additional persistence mechanisms — scheduled tasks, services, startup folders
·	Review all Type 3 logons from vboxuser across the entire environment for scope of lateral movement
·	Isolate affected host pending full forensic investigation

8. ROOT CAUSE
An attacker with access to the host established persistence via a registry run key designed to blend in with legitimate Windows components. The attacker then used WinRM to authenticate remotely using explicit credentials, suggesting stolen credential use for lateral movement. The hidden PowerShell execution flag indicates deliberate evasion of user-level detection.

9. RECOMMENDATIONS
·	Restrict registry write access to HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Run via Group Policy
·	Disable WinRM on endpoints that do not require remote management
·	Alert on any new registry run key creation outside approved change windows
·	Implement Privileged Access Workstations (PAWs) to limit lateral movement risk
·	Deploy credential guard to protect against credential theft
·	Enable PowerShell Script Block Logging and Constrained Language Mode
·	Conduct regular audits of registry run keys across the environment
