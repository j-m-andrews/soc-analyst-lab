## 1. Executive Summary

On February 24, 2026, a phishing simulation was conducted against a target user in the lab environment. The target user clicked a malicious link and submitted credentials to a fake login portal hosted at [CachyOS IP]:80. The attack was successfully detected via campaign monitoring and endpoint telemetry. No real credentials were compromised as this was a controlled lab simulation.

## 3. Attack Chain

### Initial Access — T1566 (Phishing)
Target received an email purportedly from the IT Security team with subject "Urgent: Your password expires today." The email contained a link to a credential harvesting page hosted on an attacker-controlled server.

### Credential Access — T1056 (Input Capture)
The fake login portal at `http://[CachyOS IP]:80/?rid=FVC70L2` captured the following submitted credentials:
- Username: victim@company.com
- Password: [redacted]

### Potential Impact — T1078 (Valid Accounts)
Had this been a real attack, the captured credentials could have been used for unauthorized access to corporate systems.

## 7. Containment & Remediation

In a real incident the following steps would be taken:
- Immediately reset compromised user credentials
- Block phishing domain/IP at email gateway and proxy
- Review authentication logs for any login attempts using captured credentials
- Notify affected user and conduct awareness training
- Scan endpoint for any malware dropped during the phishing visit

---

## 8. Root Cause

User clicked a phishing link due to a convincing social engineering lure using urgency around password expiration. The email bypassed simulated email gateway controls. No technical controls were in place to block access to the phishing URL at the network level.

---

## 9. Recommendations

- Deploy email authentication controls (SPF, DKIM, DMARC) to reduce phishing delivery
- Implement DNS filtering to block access to known malicious domains
- Enable MFA on all user accounts to reduce impact of credential theft
- Conduct regular phishing awareness training
- Configure Splunk alerts for bulk failed authentications following a phishing campaign
